# Fable Harness

> Claude Code（Opus / Sonnet / Haiku）を規律あるエンジニアのように働かせる、すぐ導入できる行動プロトコル——飛びつく前に確認し、前提を声に出し、大きな結論を信じる前に第三者に検証させ、本物のテストで成果を証明する。

## これは何か

Fable Harness は小さなキット——いくつかの hook、1 つの skill、複数のサブエージェント——で、Claude Code のセッションを開始するたびに自動で注入されます。Claude に新しい芸を教えるのではなく、Claude が**毎回、規律あるプロセスに従う**ことを保証します。答える前に証拠を集め、推測ではなく前提を口に出し、自分の大きな結論を信じる前に自ら反論し、変更が本当に機能することを（「良さそう」ではなく）本物の証拠で示す——これを毎回行わせます。

これはフレームワークではなく「行動の下限」だと考えてください。スプリントを計画したり CI パイプラインを回したりはしません——エージェントが作業する間、それを誠実で・慎重で・検証可能な状態に保つだけです。

## なぜ作ったのか

このキットは Fable（Anthropic の Fable モデル）の 2 回目の公開版から蒸留したもので、あのモデルが課題に取り組んだ慎重で規律ある姿勢を抽出しています。その規律を単一のモデルの中に閉じ込めておくのではなく、再利用可能なプロトコルとして取り出し、Opus（および他の Claude モデル）を取り巻く作業ハーネスを補強します。こうして、どのモデルが主導していても、セッションを重ねるごとに同じ規律で動作します。

限界も正直に述べておきます。hook と skill が移植できるのは*手順*（まず観察する、前提を述べる、結論を反対尋問する、検証の証拠を求める）だけで、モデルが生まれ持つ判断力は移植できません。しかし実際には、「良い」エージェントと「雑な」エージェントの差の大半は、判断力の欠如ではなく手順の省略から生じます。このキットが埋めるのは、まさにその差です。

## どう動くのか

- **OODA ループ**——答える前に、Claude は証拠を集め（実際のファイルを検索・読み込み、訓練記憶から推測しない）、前提を口に出し、タスクを検証可能なもの（「動けばいい」では不十分）に変換してから、小さく変更して一つずつ確認します。
- **多者対抗レビュー（adversarial review）**——このキットの目玉。大きな結論（アーキテクチャの決定、根本原因の診断、本番に影響しうるもの）を信じる前に、Claude は 3 つの独立した「反対役」サブエージェントを*並行で*派遣します。それぞれ役割が異なり、**skeptic** は論理の穴を、**red-team** はセキュリティと障害のリスクを、**simplifier** は不要な過剰設計を探します。3 つのうち過半数が「生き残った」場合にのみ、その結論は信頼されます。
- **モデルの振り分け**——推論・アーキテクチャ・根本原因の作業は現在主導しているモデルに残し、コーディングとリファクタリングは Sonnet へ、バッチのファイル処理・検索・テキスト整理は Haiku へ。仕事に合った適切なサイズのモデルを使います。
- **完了の定義（Definition of Done、TDD）**——実際のロジックに触れる変更には、自動テストと、「修正前はテストが失敗し、修正後は通過する」という証拠が必要です。出力を目で見たり、その辺の `console.log` は検証とは認めません。
- **誠実な報告**——あらゆる報告の最初の一文は実際の結果（前置きではなく）。失敗は失敗として報告し、和らげません。

## 中身

| 部品 | ファイル | 役割 |
|---|---|---|
| 行動プロトコル | `.claude/hooks/fable_protocol.md` + `inject_protocol.sh` | 各セッション開始時に注入 |
| ターンごとの一言リマインド | `.claude/hooks/prompt_nudge.sh` | ユーザーの各メッセージに一行のリマインドを注入 |
| 検証ゲート | `.claude/hooks/verify_gate.py` | このターンでコードを変更したのにテストを走らせていない場合、ターン終了を一度だけブロック（2 回目は通す） |
| 対抗レビュー | `.claude/skills/adversarial-review/` | 上記の三反対役レビューの流れを定義する skill |
| 反対役エージェント | `.claude/agents/{skeptic,red-team,simplifier}.md` | 対抗レビューで使う 3 つの独立したサブエージェントの役割 |
| モデル振り分け | `CLAUDE.md` | 上で述べた振り分け表 |
| harness 検出器 | `scripts/detect_harness.py` | 読み取り専用チェック——プロジェクトが独自の開発 harness（harnessmith、Superpowers など）を既に持っているかを確認し、あれば Fable は一歩下がって下限だけを担う |
| ガバナンス文書 | `diagnostics.md`、`model_dispatch_rules.md`、`cognitive_rubrics.md`、`future_session_letter.md` | 既知の障害モード、サブエージェント派遣テンプレート、いつ減速すべきかの基準、セッション間の引き継ぎメモ |

## クイックスタート

このリポジトリを clone し、あなたの Claude Code にこう伝えるだけです：**「INSTALL.md に従って Fable Harness をインストールして。」** Claude が自らガイドを読み、安全に（まずバックアップ、既存の設定は決して上書きしない）インストールします。具体的な内容は [INSTALL.md](INSTALL.md) を参照してください。

## バージョニング

このキットは[セマンティック バージョニング（Semantic Versioning）](https://semver.org/lang/ja/)に従い、`MAJOR.MINOR.PATCH` を用いて **1.0.0** から始まります：

- **MAJOR**——プロトコル契約の破壊的変更（hook／skill／agent の削除・改名、あるいはプロトコルの注入方法やエージェント派遣方法の非互換な変更）。ユーザーは再インストールや設定変更が必要になります。
- **MINOR**——後方互換の追加（新しい hook、skill、agent、ガバナンス規則）。既存のインストールはそのまま動き続けます。
- **PATCH**——後方互換の修正や表現の調整（hook のバグ修正、規則の明確化、誤字）。

現在のバージョンは [VERSION](VERSION) に記載し、主な変更は [CHANGELOG.md](CHANGELOG.md) に記録します。
